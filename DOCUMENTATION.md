## 1. Project Overview

TST Meet is a secure, code-based internal meeting platform. No user registration is required — participants join using unique single-use codes generated by the Admin. Real-time audio/video is powered by WebRTC with Socket.IO for signaling and event broadcasting.

### Key Design Decisions

| Decision | Reason |
|---|---|
| SQLite (not PostgreSQL) | Zero-config, single-file database — ideal for internal/on-premise deployment |
| Code-based access | No account management overhead; instant onboarding |
| NestJS backend | Structured, scalable, first-class WebSocket support |
| Next.js App Router | Server components + client interactivity |
| WebRTC mesh | Peer-to-peer video — no media server needed for small meetings |

### 3c. WebRTC P2P Video Flow

```
Participant A              Socket Server           Participant B
      │                         │                       │
      │── join-meeting ────────►│                       │
      │                         │── participant-joined ►│
      │                         │   {socketId: A}       │
      │                         │                       │
      │◄─ participant-joined ───│   B creates RTCPC     │
      │   {socketId: B}         │   and sends offer     │
      │                         │◄─ offer (A→B) ────────│
      │◄─ offer ────────────────│                       │
      │  A creates RTCPeerConn  │                       │
      │  setRemoteDescription   │                       │
      │  createAnswer           │                       │
      │── answer (A→B) ────────►│                       │
      │                         │── answer ────────────►│
      │                         │                       │
      │── ice-candidate ───────►│── ice-candidate ─────►│
      │◄─ ice-candidate ────────│◄─ ice-candidate ───────│
      │                         │                       │
      │◄══════════ Direct P2P Video/Audio Stream ══════►│
```

```
Host                    Socket Server              Target Participant
  │                          │                           │
  │── host-mute-participant ►│                           │
  │   {targetSocketId, mute}│── force-muted ───────────►│
```

### Auth Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| POST | /auth/login | None | Admin login → JWT |
| GET | /auth/me | JWT | Get current admin info |

**Login Request:**
```json
POST /api/auth/login
{
  "email": "admin@tst-meet.com",
  "password": "Admin@123456"
}
```
**Login Response:**
```json
{
  "access_token": "eyJhbGci...",
  "admin": { "id": "...", "email": "...", "name": "Super Admin" }
}
```

## 7. Feature Guide

### Admin Features

#### Creating a Meeting
1. Log in at `/admin/login`
2. Click **Create Meeting**
3. Fill in title, description, scheduled time
4. Set number of participant codes (1–50)
5. Configure permissions (chat, screen share, etc.)
6. Click **Create Meeting**
7. Copy and share codes with host and participants

#### Managing Active Meetings
- View real-time participant count
- Lock/unlock the meeting room
- End the meeting for all participants
- Click **View Details** to see live participant list, chat, codes

#### Analytics
- Total/active/ended meeting counts
- Monthly meeting bar chart
- Status breakdown with percentages
- Meeting overview table

## 8. Installation Guide (Windows)

### Prerequisites

| Requirement | Minimum | Download |
|-------------|---------|----------|
| Node.js | v18.x LTS | https://nodejs.org |
| npm | v9+ | Included with Node.js |
| Modern browser | Chrome 90+ / Edge 90+ / Firefox 88+ | — |

> **Important:** Your browser must support WebRTC. Chrome and Edge are recommended for best performance.

### Step-by-Step Installation

**Step 1 — Install Node.js**
1. Go to https://nodejs.org
2. Download the **LTS** version (18.x or later)
3. Run the installer with default settings
4. Verify: open Command Prompt and run `node --version`

**Step 2 — Extract TST Meet**
1. Extract `tst-meet.zip` to a folder, e.g. `C:\tst-meet`

**Step 3 — Install Dependencies**
1. Double-click `install.bat`
2. Wait for both backend and frontend dependencies to install
3. You'll see `[SUCCESS] All dependencies installed!`

**Step 4 — Start the Application**
1. Double-click `start.bat`
2. Two terminal windows will open (backend + frontend)
3. Your browser will open automatically to `http://localhost:3000`

### First-Time Login

1. Navigate to `http://localhost:3000/admin/login`
2. Email: `admin@tst-meet.com`
3. Password: `Admin@123456`

> **Security Note:** Change the default password in `backend/.env` before using in production.

## 9. Configuration

### Backend Environment (`backend/.env`)

```env
PORT=3001
JWT_SECRET=tst-meet-super-secret-jwt-key-2024
JWT_EXPIRY=24h
ADMIN_EMAIL=admin@tst-meet.com
ADMIN_PASSWORD=Admin@123456
FRONTEND_URL=http://localhost:3000
DATABASE_PATH=./data/tst-meet.db
```

### Frontend Environment (`frontend/.env.local`)

```env
NEXT_PUBLIC_API_URL=http://localhost:3001/api
NEXT_PUBLIC_WS_URL=http://localhost:3001
```

### Changing the Admin Password

1. Open `backend/.env`
2. Change `ADMIN_PASSWORD=YourNewPassword`
3. Delete `backend/data/tst-meet.db` (to re-seed)
4. Restart the backend

### Running on a Different Port

1. Change `PORT=3002` in `backend/.env`
2. Change `NEXT_PUBLIC_API_URL=http://localhost:3002/api` in `frontend/.env.local`
3. Change `NEXT_PUBLIC_WS_URL=http://localhost:3002` in `frontend/.env.local`
4. Restart both servers

## 11. Security Model

### Access Control

```
┌──────────────────────────────────────────────────────┐
│  ROLE          │  ACCESS METHOD    │  CAPABILITIES   │
├──────────────────────────────────────────────────────┤
│  Admin         │  Email + Password │  Full system    │
│                │  → JWT Token      │  management     │
├──────────────────────────────────────────────────────┤
│  Host          │  Host Code        │  Meeting host   │
│                │  (H-XXXXXXXX)     │  controls       │
├──────────────────────────────────────────────────────┤
│  Participant   │  User Code        │  Join + basic   │
│                │  (P-XXXXXXXX)     │  interactions   │
└──────────────────────────────────────────────────────┘
```

### Security Features

| Feature | Implementation |
|---------|----------------|
| Password hashing | bcryptjs (10 rounds) |
| Admin sessions | JWT (24h expiry) |
| Meeting access | Unique 8-char alphanumeric codes |
| Code expiry | 7 days after creation |
| Code deactivation | All codes deactivated when meeting ends |
| Meeting locking | Host can block new participants |
| Forced removal | Host can disconnect any participant |
| CORS | Restricted to configured frontend URL |

### Code Format

- Host codes: `H-XXXXXXXX` (e.g. `H-K7MN2QRP`)
- Participant codes: `P-XXXXXXXX` (e.g. `P-XJ3BVW9F`)
- Character set: `ABCDEFGHJKLMNPQRSTUVWXYZ23456789` (no ambiguous chars like 0/O, 1/I)

## 12. Troubleshooting

### Backend won't start

**Problem:** `Cannot find module` errors  
**Fix:** Run `install.bat` again from the `tst-meet` folder

**Problem:** Port 3001 already in use  
**Fix:** Change `PORT=3002` in `backend/.env` and update frontend `.env.local`

**Problem:** Database errors  
**Fix:** Delete `backend/data/tst-meet.db` and restart — it will auto-recreate

---

### Frontend won't start

**Problem:** Port 3002 already in use  
**Fix:** Edit `start.bat` — change `npm run dev` to `npm run dev`

**Problem:** "Cannot connect to API"  
**Fix:** Ensure backend is running at port 3001; check `frontend/.env.local`

---

### Video/Audio Issues

**Problem:** Camera/mic not working  
**Fix:**
1. Check browser permissions (click the lock icon in address bar)
2. Allow camera and microphone access
3. Ensure no other app is using the camera

**Problem:** Can't see other participants' video  
**Fix:**
1. Ensure both users are on the same network or reachable via internet
2. WebRTC requires direct peer connectivity; corporate firewalls may block it
3. Try using Chrome or Edge for best WebRTC support

**Problem:** Screen share not working  
**Fix:** Ensure "Allow Screen Share" is enabled in meeting settings (Admin → Meeting Details)

---

### Meeting Codes

**Problem:** "Invalid or expired code"  
**Fix:**
1. Check the code is typed correctly (no spaces, uppercase)
2. Ensure the meeting is not ended or cancelled
3. Ensure the code hasn't expired (7-day limit)
4. Generate new codes in Admin → Meeting Detail → Codes tab

---

### Admin Login

**Problem:** "Invalid credentials"  
**Fix:** Default credentials are `admin@tst-meet.com` / `Admin@123456`  
If changed, check `backend/.env`

**Problem:** Logged out unexpectedly  
**Fix:** JWT tokens expire after 24 hours — simply log in again
